Ramin Yazdani | pymol-hydrophobicity-tools | main | WIP(extension): Add color palette parameter to hydrophobicity function

⚠️ OOPS: Added palette parameter but forgot to validate it!

Enhanced the extension with color palette support, but overlooked input validation.

What I implemented:
- Added palette parameter to hydrophobicity() function signature
- Palette will be used with PyMOL's spectrum command for color gradients
- Intended to support PyMOL palettes like blue_white_red, rainbow, etc.
- Parameter gets passed directly to PyMOL coloring commands

What's wrong:
No validation that the palette name is actually a valid PyMOL palette. Users 
can pass any string (e.g., "rainbow2", "blue_yellow", "custom_palette") and 
the extension will accept it, then fail later with a cryptic PyMOL error:
  Error: Unknown palette 'rainbow2'
  
The error message doesn't clearly explain what palettes ARE valid or how to 
fix the issue.

How I noticed:
While testing different palette names, I mistyped "blue_white_green" as 
"blue_green" and got a confusing error message from PyMOL's internal code. 
Realized users would face the same frustration without knowing valid options.

Impact:
Poor user experience - users get cryptic errors instead of helpful messages. 
Without validation, there's no way to know which palette names are valid 
without consulting PyMOL documentation.
